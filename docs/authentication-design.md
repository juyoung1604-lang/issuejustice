# 회원 인증 방식 설계 문서

## 1. 인증 정책 기본 원칙
- 1인 1계정 원칙
- 개인정보 최소 수집
- 익명 활동 허용 (닉네임 기반)
- 추천 및 댓글 악용 방지
- 운영 안정성을 위한 책임성 확보

## 2. 회원 유형
- 일반 사용자
  - 이슈 열람
  - 댓글 작성
  - 추천 참여
- 인증 사용자
  - 휴대폰 인증 완료 사용자
  - 이슈 등록 가능
  - 추천 가능
- 관리자
  - 이슈 승인/거절
  - 댓글 관리
  - 사용자 차단
  - 추천 모니터링

## 3. 회원 인증 방식 (3단계)
### 1단계: 소셜 로그인
- 카카오, 구글, 이메일 로그인 지원

### 2단계: 휴대폰 인증
- 추천 및 이슈 등록 시 필수

### 3단계: 다중 계정 방지
- IP 및 기기 검증 적용
- 이상 패턴 감지 시 추가 인증 또는 제한

## 4. 회원가입 흐름
1. 사용자 접속
2. 소셜 로그인
3. 닉네임 설정
4. 휴대폰 인증
5. 가입 완료

## 5. 로그인 구조
- JWT 기반 인증
- Access Token: 2시간 유효
- Refresh Token: 30일 유효

## 6. 추천 시스템 인증 정책
- 추천은 인증 사용자만 가능
- 이슈당 1회 추천
- 로그인 및 휴대폰 인증 확인 후 추천 허용

## 7. 댓글 시스템 인증 정책
- 로그인 사용자 댓글 가능
- 권장 정책: 휴대폰 인증 사용자만 댓글 허용
- 댓글 스팸 방지를 위한 속도 제한 적용

## 8. 익명성 보호 정책
- 활동은 닉네임 기반으로 표시
- 이메일, 전화번호, 실명 등 개인정보는 공개하지 않음

## 9. 계정 악용 방지 정책
- `phone_hash` 중복 차단
- IP 중복 감지
- Device fingerprint 적용
- 추천 속도 제한 및 스팸 방지

## 10. 사용자 차단 정책
- 관리자는 아래 권한을 가짐
  - 댓글 차단
  - 추천 차단
  - 계정 정지

## 11. 개인정보 보호 정책
- 수집 정보
  - 이메일
  - 닉네임
  - `phone_hash`
- 수집 금지 정보
  - 실명
  - 주소
  - 주민등록번호

## 12. 권장 인증 서비스
- 카카오 로그인
- 토스 인증
- NICE 본인인증

## 13. MVP 권장 인증 구조
- 카카오 로그인 + 휴대폰 인증

## 데이터 모델 권장 필드

### `profiles`
- `id` (auth uid)
- `nickname`
- `phone_hash` (unique)
- `is_phone_verified` (boolean)
- `is_admin` (boolean)
- `created_at`

### 보안 로그(권장)
- `auth_audit_logs`
  - `user_id`
  - `ip_hash`
  - `device_fingerprint`
  - `event_type` (login, verify_phone, recommend, comment)
  - `created_at`

## 운영 체크포인트
- 휴대폰 인증 미완료 사용자의 추천/이슈 등록 API 차단
- 추천 API에 이슈당 1회 유니크 제약 유지
- 댓글/추천에 rate limit 적용
- 관리자 콘솔에서 사용자 제재 이력 조회 가능하게 설계
